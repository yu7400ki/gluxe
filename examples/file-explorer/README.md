# file-explorer

A two-pane file explorer example that exercises the current gluxe API:

- **Left pane** — navigable directory listing (click a folder to enter, "↑ Up" to go back).
- **Right pane** — text preview of the selected file (`fs.readTextFile`).

This example was built to find the boundaries of what is possible today.
Every gap discovered during development is documented in the **Roadmap** below.

---

## How to run

```sh
# 1. Install JS dependencies (from the repo root)
pnpm install

# 2. Bundle JS
cd examples/file-explorer
pnpm build       # produces dist/assets/<entry>-<hash>.js + gluxe.manifest.json

# 3. Build & launch
cargo run        # first build is slow (compiles GPUI from source)
```

Or via the CLI from the repo root:

```sh
gluxe run --project examples/file-explorer
```

## What is implemented

Using **only the current public API** (no framework changes):

| Feature                       | How                                                                                                                                                                                                                                                                                                                                                                                                                       |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Directory listing             | `fs.readDir(path)` → sorted dirs-first                                                                                                                                                                                                                                                                                                                                                                                    |
| Navigate into folder          | `onClick` → `fs.canonicalize(path)` → `setCwd(canonical)`                                                                                                                                                                                                                                                                                                                                                                 |
| Navigate up                   | "↑ Up" button → `fs.canonicalize(cwd + "/..")` (stops at root)                                                                                                                                                                                                                                                                                                                                                            |
| Text file preview             | `fs.readTextFile(path)` on file click                                                                                                                                                                                                                                                                                                                                                                                     |
| Error display                 | `try/catch` around every `fs` call                                                                                                                                                                                                                                                                                                                                                                                        |
| Hover highlight               | `_hover: { backgroundColor }` style prop                                                                                                                                                                                                                                                                                                                                                                                  |
| Selected highlight            | `backgroundColor` set when `selected === path`                                                                                                                                                                                                                                                                                                                                                                            |
| Dark colour scheme            | `backgroundColor` / `color` style props                                                                                                                                                                                                                                                                                                                                                                                   |
| Pane border                   | `borderWidth` / `borderColor` style props                                                                                                                                                                                                                                                                                                                                                                                 |
| Directory bold                | `fontWeight: "bold"` style prop                                                                                                                                                                                                                                                                                                                                                                                           |
| Pointer cursor                | `cursor: "pointer"` style prop                                                                                                                                                                                                                                                                                                                                                                                            |
| Long-name clipping            | `whiteSpace: "nowrap"` + `textOverflow: "ellipsis"` style props                                                                                                                                                                                                                                                                                                                                                           |
| Scrollable panes              | `overflowY: "scroll"` style prop on both panes                                                                                                                                                                                                                                                                                                                                                                            |
| TextInput — filter bar        | `<TextInput value={filter} onChangeText={setFilter} placeholder="Filter…" />` live-filters the directory listing                                                                                                                                                                                                                                                                                                          |
| TextInput — controlled value  | `value` prop drives content; echo-safe: external `setState` resets caret position                                                                                                                                                                                                                                                                                                                                         |
| TextInput — full editing      | Backspace/Delete/Home/End, Ctrl+A/C/V/X, drag-selection, arrow-key navigation                                                                                                                                                                                                                                                                                                                                             |
| TextInput — IME               | `EntityInputHandler` / `replace_and_mark_text_in_range` — underlined composition, CJK etc.                                                                                                                                                                                                                                                                                                                                |
| TextInput — onChangeText      | Called with the new string on every keystroke (React Native–style)                                                                                                                                                                                                                                                                                                                                                        |
| TextInput — onSubmit          | Called with the current string when the user presses Enter                                                                                                                                                                                                                                                                                                                                                                |
| TextInput — onFocus / onBlur  | Dispatched when the field gains or loses keyboard focus                                                                                                                                                                                                                                                                                                                                                                   |
| Fill / stretch sizing         | `RootView` renders full-window without centering (`items_start` + `justify_start`); root container uses `width: "100%"` / `height: "100%"`                                                                                                                                                                                                                                                                                |
| `alignItems` full support     | `"flex-start"` / `"center"` / `"flex-end"` / `"baseline"` / `"stretch"` all mapped                                                                                                                                                                                                                                                                                                                                        |
| `justifyContent` full support | `"flex-start"` / `"center"` / `"flex-end"` / `"space-between"` / `"space-around"` / `"space-evenly"` all mapped                                                                                                                                                                                                                                                                                                           |
| Absolute start path           | `fs.cwd()` on mount replaces `"."` with the real absolute working directory                                                                                                                                                                                                                                                                                                                                               |
| Canonical navigation          | `fs.canonicalize(path)` called on every navigate; `..` resolves cleanly and stops at the filesystem root                                                                                                                                                                                                                                                                                                                  |
| `readDir` size + modified     | `DirEntry` now includes `size` (bytes) and `modified` (epoch ms); returned by the Rust plugin alongside existing fields                                                                                                                                                                                                                                                                                                   |
| Status bar                    | When a file is selected `fs.metadata(path)` is fetched and the status bar shows human-readable size and last-modified date                                                                                                                                                                                                                                                                                                |
| `fs.metadata`                 | `{ isDir, isFile, isSymlink, size, readonly, modified, accessed, created }` — uses `symlink_metadata` so `isSymlink` is accurate                                                                                                                                                                                                                                                                                          |
| `fs.cwd`                      | Returns the process's absolute working directory                                                                                                                                                                                                                                                                                                                                                                          |
| `fs.homeDir`                  | Returns the current user's home directory                                                                                                                                                                                                                                                                                                                                                                                 |
| `fs.canonicalize`             | Returns an absolute, normalised path; strips Windows `\\?\` verbatim prefix                                                                                                                                                                                                                                                                                                                                               |
| File-type icons               | `import folderIcon from "./src/icons/folder.svg"` + `<Image src={folderIcon} style={{ width: 16, height: 16 }} />` — the gluxe Vite plugin emits SVGs as hashed files and prefixes their bundled URLs with `asset://`; GPUI's `svg_renderer` decodes and draws them                                                                                                                                                       |
| Image file preview            | Selecting a file with a known image extension (png/jpg/jpeg/gif/webp/bmp/ico/svg) renders it in the right pane via `<Image src={absolutePath} style={{ width: "100%", height: "100%" }} />`; GPUI's `ObjectFit::Contain` default fits the picture to the pane preserving aspect ratio; binary data is never read as text                                                                                                  |
| Keyboard navigation           | Entry list receives `autoFocus` on launch; `onKeyDown` on the list container drives a cursor index. Key map: ↓/↑ = move cursor, Home/End = jump to start/end, PageDown/PageUp = ±10, Enter = open file or enter directory, Backspace = go up a directory, Esc = clear filter. A blue border ring highlights the cursor row. Clicking the list re-focuses it; clicking the filter input focuses that instead.              |
| `onKeyDown` event             | Generic keyboard event added to the gluxe-core framework (`EventProps.onKeyDown`). Any `<View>` (or `<Image>`) with `onKeyDown` or `autoFocus` becomes keyboard-focusable: GPUI wires a `FocusHandle` + `track_focus` + `on_key_down`. Click-to-focus is automatic (GPUI's `track_focus` binds focus on mouse-down). `GpuiKeyboardEvent.key` carries the GPUI key name; `shift`/`ctrl`/`alt`/`meta` carry modifier state. |
| `autoFocus` prop              | `<View autoFocus>` receives keyboard focus immediately on first mount (fires `window.focus()` once, guarded so re-renders don't steal focus back).                                                                                                                                                                                                                                                                        |

## Roadmap — features the framework still needs

Discovered while building this example. Items marked **GPUI-ready** mean GPUI
already exposes the required API at the pinned rev; only framework-side exposure
work is needed. Items without that label require genuine new design.

---

### Future / Phase 4+

| Feature                              | Notes                                                                                                                                                                                                                    |
| ------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Scroll-into-view for keyboard cursor | No JS-side scroll API exists today — the `ScrollHandle` is Rust-only. The keyboard-cursor row can scroll out of view when navigating a long directory with ↑/↓. Needs a new `scrollTo` plugin or a direct JS scroll API. |
| Context menu (right-click)           | Needs `onContextMenu` event + overlay rendering                                                                                                                                                                          |
| Drag-and-drop                        | Requires new GPUI drag event bridge                                                                                                                                                                                      |
| Multi-select                         | State-only (no framework changes); depends on keyboard navigation (now done)                                                                                                                                             |
| File operations UI                   | Create/rename/delete — `fs.mkdir`, `fs.remove`, `fs.rename` (rename not yet in plugin)                                                                                                                                   |
| Breadcrumb nav                       | Split `fs.canonicalize` result on path separator into segments                                                                                                                                                           |
